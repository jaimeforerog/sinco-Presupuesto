# Follow-ups

Seguimiento de deuda técnica y trabajos pendientes generados por reviews de slices. Se cierra cada ítem cuando el slice que lo resuelve pasa review.

**Convención:**
- Un ítem por línea, formato `- [ ] #{N} — {título corto} — origen: slice-{N}, review §{sección}. Propuesta: {acción sugerida}.`
- Se marca `[x]` cuando el slice resolvente cierra review con `approved`.

- [x] #23 — bug serialización `Moneda` con System.Text.Json — origen: descubierto por el visor de eventos (`slices/_obs-visor-eventos/`). Al deserializar eventos desde el stream, `Moneda.Codigo` quedaba `null` porque STJ usaba el constructor default del struct (propiedad get-only sin setter). El JSON almacenado en Postgres era correcto; el problema era la rehidratación C#. No manifestaba errores observables porque las proyecciones son inline (escritas con el evento en memoria antes de persistir). **Cerrado 2026-04-24** — añadido `[JsonConstructor]` a `Moneda(string codigo)` + 3 tests de round-trip STJ en Slice00 (§6.23 — round-trip preserva `Codigo`, deserialización desde JSON PascalCase con `PropertyNameCaseInsensitive`, Dinero round-trip preserva Valor y Moneda). Estilo TDD al revés (METHODOLOGY §7.3): tests rojos que reproducen el bug → fix → verde.
